# チャットアプリ (ChatFourYou)

PythonとMySQLを使ったリアルタイムチャットアプリケーション

## 機能

- ✅ 複数人でチャット可能
- ✅ メッセージの削除機能
- ✅ リアルタイム同期（Socket.IO使用）
- ✅ ユーザー管理システム

## 技術スタック

- **バックエンド**: Flask + Flask-SocketIO
- **データベース**: MySQL
- **フロントエンド**: HTML, CSS, JavaScript
- **リアルタイム通信**: Socket.IO

## セットアップ手順

### 1. 必要なパッケージのインストール

```bash
pip install -r requirements.txt
```

### 2. データベースの作成

MySQLサーバーにログインして、`schema.sql`を実行してください：

```bash
mysql -h 172.16.2.26 -u root -p < schema.sql
```

パスワード: `ae21215926`

### 3. 設定ファイルの編集（必要に応じて）

`config.py`を開いて、データベース設定を確認してください：

```python
DB_CONFIG = {
    'host': '172.16.2.26',
    'user': 'root',  # 自分のユーザーIDに変更
    'password': 'ae21215926',
    'database': 'chat_app',
    'charset': 'utf8mb4'
}
```

### 4. アプリケーションの起動

```bash
python app.py
```

サーバーが起動したら、ブラウザで以下にアクセス：
```
http://localhost:5000
```

## プロジェクト構造

```
ChatFourYou/
├── app.py              # メインアプリケーション
├── database.py         # データベース操作
├── config.py           # 設定ファイル
├── schema.sql          # データベーススキーマ
├── requirements.txt    # 必要なパッケージ
├── templates/
│   └── index.html      # メインページ
└── static/
    ├── style.css       # スタイルシート
    └── script.js       # クライアント側JavaScript
```

## 使い方

1. アプリケーションを起動
2. ユーザー名を入力してログイン
3. メッセージを入力して送信
4. 他のユーザーとリアルタイムでチャット
5. メッセージの削除ボタンで不要なメッセージを削除可能

## トラブルシューティング

### データベース接続エラー
- MySQLサーバーが起動しているか確認
- `config.py`の接続情報が正しいか確認
- データベース`chat_app`が作成されているか確認

### Socket.IO接続エラー
- ファイアウォールがポート5000をブロックしていないか確認
- ブラウザのコンソールでエラーメッセージを確認
